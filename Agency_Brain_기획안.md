# Agency Brain: AI 기반 프로젝트 지식 관리 & 제안 전략 통합 플랫폼

## Executive Summary

Agency Brain은 웹에이전시의 **모든 프로젝트 지식 자산**(제안서, 기획서, 개발 문서, 운영 매뉴얼 등)을 Graph DB 기반으로 구조화하고, 직원 누구나 실시간으로 과거 경험과 노하우를 검색·활용할 수 있는 **AI 기획 도구**입니다.

단순한 문서 검색을 넘어, **프로젝트 수주 → 기획 → 개발 → 운영** 전 단계에서 AI가 적절한 레퍼런스와 전략을 제안하여, 업무 효율을 50% 이상 향상시키고 수주율을 높이는 것을 목표로 합니다.

---

## 1. 배경 및 목적

### 1.1 현재의 문제점

**지식 자산의 분산 및 사장(死藏)**
- 수백 개 프로젝트의 기획서, 제안서, 개발 문서가 개인 PC나 공유 드라이브에 흩어져 있음
- 신규 입사자는 과거 프로젝트를 알 수 없고, 베테랑 직원의 경험은 문서화되지 않음
- "예전에 비슷한 거 했던 것 같은데..." 하며 찾다가 시간 낭비

**반복 작업의 비효율**
- 매번 새로운 프로젝트마다 IA, 기능정의서, 테스트 시나리오를 처음부터 작성
- 유사한 커머스 프로젝트인데도 결제/배송 정책을 다시 정리
- 제안서 작성 시 과거 성공 사례를 찾지 못해 설득력 부족

**의사결정의 불확실성**
- "이 기능 개발에 얼마나 걸릴까?" → 감으로 예측
- "Cafe24 + ERP 연동 가능할까?" → 개발팀에게 물어봐야 알 수 있음
- "과거 비슷한 프로젝트 견적이 얼마였지?" → 찾을 수 없음

### 1.2 목적

1. **프로젝트 지식의 자산화**: 흩어진 문서들을 구조화된 지식 그래프로 전환
2. **업무 효율 극대화**: 반복 작업 자동화 및 레퍼런스 즉시 제공으로 생산성 향상
3. **수주 경쟁력 강화**: 과거 성공 사례 + 최신 트렌드 결합으로 차별화된 제안서 작성
4. **의사결정 지원**: 데이터 기반 견적, 리스크, 리소스 예측
5. **지식 전승**: 베테랑의 암묵지를 신규 직원에게 전달

## 2. 시스템 개요

### 2.1 핵심 개념

Agency Brain은 **"모든 프로젝트 문서가 곧 학습 데이터"**라는 전제 하에 설계되었습니다.

**학습 대상 문서 (Input)**
- **제안 단계**: RFP, 제안서, 견적서, PT 자료
- **기획 단계**: 요구사항정의서, IA/메뉴구조도, 화면설계서, 기능정의서, Usecase, 정책서
- **개발 단계**: WBS, 기술 스펙 문서, API 명세서, 개발 가이드, 테스트 시나리오
- **디자인 단계**: 디자인 가이드, 스타일 가이드, 컴포넌트 라이브러리, 시안 이미지
- **운영 단계**: 운영 매뉴얼, 장애 리포트, 유지보수 히스토리, 성과 리포트

**출력 (Output)**
- 직원이 질문하면 적절한 레퍼런스 문서, 이미지, 전략을 즉시 제공
- 새로운 프로젝트 시 자동으로 문서 초안 생성 (IA 템플릿, 기능정의서 초안 등)

### 2.2 전방위 파일 형식 지원 (Universal Document Processing)

**핵심 원칙: "어떤 형식이든 학습 가능해야 한다"**

웹에이전시에서 사용되는 모든 파일 형식을 빠짐없이 처리할 수 있어야 합니다. 파일 확장자에 관계없이 그 안의 지식을 추출하고 구조화하는 것이 핵심입니다.

#### OCR 및 특수 처리

**이미지화된 PDF 처리 (OCR)**
- Tesseract OCR: 한글/영문 동시 인식
- AWS Textract: 표/양식 정확도 높음
- Google Cloud Vision API: 손글씨 인식 가능
- **적용 예시**: 스캔된 계약서, 손글씨 메모, 회의록 사진

**손글씨/화이트보드 처리**
- 회의 중 화이트보드 사진 → 텍스트 추출
- 기획 회의 메모 → 아이디어 DB화

**동영상 내 텍스트 추출**
- 프레젠테이션 영상 → 슬라이드 텍스트 OCR
- 시연 영상 → UI 텍스트 인식
- 자막 추출 → 콘텐츠 이해

#### 파일 처리 파이프라인

```
[파일 수집]
    ↓
[확장자 감지]
    ↓
[적절한 파서 선택]
    ↓
[텍스트/이미지/메타데이터 추출]
    ↓
[전처리 (노이즈 제거, 정규화)]
    ↓
[임베딩 (텍스트 → Vector, 이미지 → CLIP)]
    ↓
[Graph DB 저장 (노드/관계 생성)]
    ↓
[검색 인덱싱]
```

#### 버전 관리 및 중복 처리

**동일 문서의 버전 관리**
- `제안서_v1.0.docx`, `제안서_v2.3_최종.docx`, `제안서_최최종진짜마지막.docx`
- → AI가 버전 히스토리를 파악하고 **최신 버전 우선 제공**, 변경 이력 추적 가능

**중복 파일 자동 감지**
- 파일명은 다르지만 내용이 99% 동일 → 중복으로 처리
- "같은 문서가 5개 폴더에 흩어져 있어요" → 하나로 통합 관리

### 2.3 차별화 포인트

| 기존 솔루션 | Agency Brain |
|---|---|
| 키워드 검색 (파일명, 제목) | 의미 기반 검색 (Graph DB) |
| 문서 단위 검색 | 문단, 이미지, 개념 단위 정밀 검색 |
| 과거 데이터만 활용 | 과거 데이터 + 실시간 외부 트렌드 |
| 텍스트만 검색 | 텍스트 + 이미지 + 동영상 (Multi-modal) |
| 특정 파일 형식만 지원 | **모든 파일 형식 지원 (Universal)** |
| 문서 찾기만 가능 | 문서 생성 + 전략 제안 + 리스크 예측 |

## 3. 핵심 아키텍처

### 3.1 Knowledge Graph 구조

프로젝트 정보를 **노드(Node)**와 **관계(Edge)**로 구조화하여, 단순 키워드 검색보다 훨씬 정교한 지식 탐색이 가능합니다.

#### 노드(Node) 정의

| 노드 유형 | 설명 및 예시 | 주요 활용처 |
|---|---|---|
| **Project** | 프로젝트 기본 정보 (프로젝트명, 기간, 고객사) | 전체 맥락 파악 |
| **Industry** | 고객사 산업군 (패션, 뷰티, F&B, 공공, 금융, 헬스케어) | 유사 업종 레퍼런스 탐색 |
| **Concept** | 디자인/UX 컨셉 (미니멀, 레트로, 다크모드, 인터랙티브) | **최우선** 검색 기준 |
| **Platform** | 구축 플랫폼 (Cafe24, Shopify, Headless, AWS, 자체구축) | 기술 스택 호환성 판단 |
| **Target** | 타겟 오디언스 (MZ세대, 액티브 시니어, B2B 구매자) | 타겟별 UX 전략 |
| **Feature** | 구현 기능 (회원가입, 결제, 라이브커머스, AI 챗봇) | 기능 단위 재사용 |
| **Tech_Stack** | 기술 스택 (React, Next.js, Node.js, PostgreSQL) | 개발자 기술 참조 |
| **Conversion** | 목표 전환 행동 (구매, 가입, 상담신청, 브랜딩) | KPI 설정 근거 |
| **Pain_Point** | 고객 고민 (로딩 속도, 이탈률, 모바일 가독성) | 문제 해결 전략 도출 |
| **Risk** | 과거 프로젝트 리스크 (일정 지연, 기술 이슈) | 사전 위험 예방 |
| **Cost** | 견적 정보 (총 비용, 공수, 인력 구성) | 견적 자동화 |
| **Performance** | 성과 지표 (전환율, 이탈률, 페이지뷰, SEO 순위) | ROI 증명 자료 |
| **Document** | 문서 종류 (제안서, IA, 기능정의서, 테스트 시나리오) | 문서 유형별 검색 |
| **File** | 실제 파일 정보 (파일명, 경로, 형식, 버전) | **파일 직접 접근** |
| **Image** | 디자인 시안, UI 이미지 | 비주얼 레퍼런스 |
| **Video** | 시연 영상, 프레젠테이션 동영상 | 동적 컨텐츠 |
| **Code_Snippet** | 코드 조각, 함수, API | 개발 재사용 |
| **Regulation** | 관련 법규 (개인정보보호법, 전자상거래법, 의료광고법) | 컴플라이언스 체크 |

#### 관계(Edge) 예시

```
(프로젝트 A) - [BELONGS_TO] -> (Industry: 뷰티)
(프로젝트 A) - [APPLIED_CONCEPT] -> (Concept: 미니멀리즘)
(프로젝트 A) - [BUILT_ON] -> (Platform: Cafe24)
(프로젝트 A) - [USES_TECH] -> (Tech: React, Node.js)
(프로젝트 A) - [IMPLEMENTED] -> (Feature: 라이브커머스)
(프로젝트 A) - [INCREASED] -> (Performance: 구매전환율 20% 향상)
(프로젝트 A) - [HAD_RISK] -> (Risk: 재고 연동 5분 지연)
(프로젝트 A) - [COST] -> (Cost: 3,500만원, 8주, 5명)
(프로젝트 A) - [HAS_DOCUMENT] -> (Document: 기능정의서)
(Document: 기능정의서) - [STORED_AS] -> (File: 기능정의서_v2.3.pdf)
(File: 기능정의서_v2.3.pdf) - [CONTAINS_IMAGE] -> (Image: IA구조도.png)
(Feature: 결제) - [COMPATIBLE_WITH] -> (Platform: Shopify)
(Platform: Cafe24) - [REQUIRES] -> (Tech: PHP, MySQL)
(Code_Snippet: 결제모듈) - [WRITTEN_IN] -> (Tech: JavaScript)
```

### 3.2 Multi-modal RAG (텍스트 + 이미지 + 동영상 + 코드)

웹에이전시는 **비주얼이 설득력**입니다. 텍스트뿐 아니라 이미지, 동영상, 코드까지 검색 가능해야 합니다.

**텍스트 검색 (Semantic Search)**
- 의미 기반 검색: "결제 취소 프로세스" → 과거 문서의 결제 관련 섹션 정확히 추출
- 다국어 지원: 한글/영문 혼용 문서도 정확히 검색

**이미지 검색 (CLIP 모델 활용)**
- "어두운 배경의 고급스러운 메인 시안" → AI가 과거 제안서/보고서에서 해당 스타일 이미지 검색
- "플로팅 장바구니 UI 디자인" → 실제 구현 화면 캡처 이미지 반환
- PSD, AI, Figma 파일 내부 레이어도 검색 가능

**동영상 검색 (Video Scene Understanding)**
- "인터랙티브 애니메이션 화려한 레퍼런스" → 과거 프로젝트 시연 영상에서 해당 장면 캡처

**목적:** "당신들이 만든 사이트가 실제로 성과가 나왔나요?"라는 질문에 명확한 답변.

**저장 항목:**
- 프로젝트 론칭 전/후 3개월 비교 데이터
  - 구매전환율, 이탈률, 페이지뷰, 재방문율, SEO 순위, 평균 체류 시간

**활용:**
```

→ "D프로젝트: 모바일 이탈률 68% → 42% 감소 (론칭 6주차)"
→ "E프로젝트: 모바일 구매전환율 1.2% → 2.8% 증가 (론칭 3개월)"
```

**제안서 적용:**
- [예상 효과] 섹션에 "유사 프로젝트 성과 데이터 기반" 그래프 자동 생성

**제안서 자동 삽입:**

## 10. 결론

Agency Brain은 단순한 "문서 검색 도구"가 아닙니다. 

웹에이전시의 **모든 프로젝트 지식**을 **모든 파일 형식**으로부터 추출하여 AI로 구조화하고, 직원 누구나 과거의 성공과 실패를 즉시 학습하여 더 나은 결과물을 만들어내는 **지능형 업무 플랫폼**입니다.

**핵심 차별점:**
- ✅ PDF, DOCX부터 HWP, PSD, Figma, 영상, 음성까지 **모든 파일 형식 지원**
- ✅ 텍스트, 이미지, 동영상, 코드 모두 검색 가능한 **Multi-modal RAG**
- ✅ 과거 지식 + 최신 트렌드 결합
- ✅ 제안서 작성부터 기획, 개발, 운영까지 **전 단계 지원**

**"과거에 우리가 했던 모든 일"**이 **"미래의 경쟁력"**이 됩니다.

**"어떤 형식이든, 어디에 있든, 누가 만들었든"** 모든 지식을 찾아내고 활용할 수 있는 시스템.
