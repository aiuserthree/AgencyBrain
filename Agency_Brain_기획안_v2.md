# Agency Brain: AI 기반 프로젝트 지식 관리 플랫폼 (현실적 실행 버전)

## Executive Summary

Agency Brain은 웹에이전시의 **핵심 프로젝트 자산**(제안서, 기획서, 개발 문서)을 AI로 구조화하고, **실시간 검색 및 재사용**을 가능하게 하는 실무 중심 플랫폼입니다.

**핵심 차별점:**
- 구글 Workspace/Vertex AI에는 없는 **웹에이전시 특화 기능**
- 과거 프로젝트 기반 **견적 자동 산출** 및 **수주율 분석**
- 프로젝트 메타데이터 기반 **정밀 검색** (업종, 기술스택, 고객사별)
- **실제 구현 가능한** 범위로 설계

---

## 1. 배경 및 목적

### 1.1 현재의 문제점

**지식 자산의 분산 및 사장**
- 수백 개 프로젝트의 문서가 개인 PC, 공유 드라이브에 흩어짐
- 신규 입사자는 과거 프로젝트를 찾을 수 없음
- "예전에 비슷한 거 했던 것 같은데..." 하며 시간 낭비

**반복 작업의 비효율**
- 매번 IA, 기능정의서, 테스트 시나리오를 처음부터 작성
- 유사한 커머스 프로젝트인데도 결제/배송 정책을 다시 정리
- 제안서 작성 시 과거 성공 사례를 찾지 못함

**의사결정의 불확실성**
- "이 기능 개발에 얼마나 걸릴까?" → 감으로 예측
- "과거 비슷한 프로젝트 견적이 얼마였지?" → 찾을 수 없음

### 1.2 목적

1. **프로젝트 지식의 자산화**: 문서를 구조화된 지식으로 전환
2. **업무 효율 극대화**: 반복 작업 50% 단축
3. **수주 경쟁력 강화**: 데이터 기반 제안서 작성
4. **의사결정 지원**: 과거 데이터 기반 견적/리소스 예측

## 2. 시스템 개요

### 2.1 핵심 개념

**학습 대상 문서 (우선순위 순)**

| 순위 | 문서 종류 | 파일 형식 | 비고 |
|---|---|---|---|
| 1순위 | 제안서, 견적서 | PDF, DOCX, HWP, PPTX | **수주 직결** |
| 2순위 | 기획서 (IA, 기능정의서, 화면설계서) | DOCX, HWP, XLSX, PDF | **재사용 빈도 높음** |
| 3순위 | 디자인 시안 | JPG, PNG, PDF | **비주얼 레퍼런스** |
| 4순위 | 개발 문서 (API 명세, 코드) | MD, TXT, JS, PY | **기술 참조** |
| 5순위 | 운영 매뉴얼, 보고서 | DOCX, PDF | **유지보수 참조** |

**현실적 파일 처리 범위**
- ✅ **Microsoft Office**: DOCX, PPTX, XLSX (python-docx, python-pptx, openpyxl)
- ✅ **PDF**: 텍스트 추출 + OCR (pdfplumber, Tesseract)
- ✅ **한글(HWP)**: 텍스트 추출 (pyhwp, olefile)
- ✅ **이미지**: JPG, PNG (Pillow + CLIP 임베딩)
- ✅ **코드**: JS, PY, HTML, CSS (텍스트 파싱)
- ⚠️ **Figma/Sketch**: API 연동 (Phase 4 이후)
- ❌ **PSD 레이어 검색**: 기술적 어려움 → 제외
- ❌ **동영상 Scene Understanding**: 비용 과다 → 제외

### 2.2 구글 엔터프라이즈와의 차별점

| 항목 | 구글 Workspace/Vertex AI | Agency Brain |
|---|---|---|
| **검색 방식** | 키워드 + 시맨틱 검색 | **프로젝트 메타데이터 기반 정밀 검색** |
| **데이터 구조** | 플랫 문서 검색 | **Graph DB (프로젝트-문서-파일 관계)** |
| **견적 산출** | ❌ 없음 | **✅ 과거 프로젝트 기반 자동 견적** |
| **수주율 분석** | ❌ 없음 | **✅ Win/Loss 패턴 학습** |
| **업종 특화** | 범용 | **✅ 웹에이전시 전용 (Industry, Platform, Tech)** |
| **제안서 생성** | 일반 문서 작성 지원 | **✅ 과거 성공 제안서 기반 자동 초안** |
| **고객사 DB** | ❌ 없음 | **✅ 고객사별 의사결정 패턴 학습** |
| **기획 자동화** | ❌ 없음 | **✅ IA, 기능정의서 자동 초안** |

## 3. 핵심 아키텍처

### 3.1 Knowledge Graph 구조

**노드(Node) 정의 - 실무 중심**

| 노드 유형 | 설명 | 활용처 |
|---|---|---|
| **Project** | 프로젝트명, 기간, 고객사, 수주금액 | 전체 맥락 |
| **Industry** | 산업군 (패션, 뷰티, F&B, 공공) | 유사 업종 레퍼런스 |
| **Platform** | 구축 플랫폼 (Cafe24, Shopify, 자체구축) | 기술 호환성 |
| **Tech_Stack** | 기술 스택 (React, Node.js, PostgreSQL) | 개발 참조 |
| **Feature** | 구현 기능 (결제, 회원가입, 라이브커머스) | 기능 단위 재사용 |
| **Cost** | 견적 정보 (총 비용, 공수, 인력) | **견적 자동화** |
| **Document** | 문서 종류 (제안서, IA, 기능정의서) | 문서 유형별 검색 |
| **File** | 실제 파일 (경로, 형식, 버전) | 파일 접근 |
| **Win_Loss** | 수주 성공/실패 여부 + 사유 | **수주율 분석** |

**관계(Edge) 예시**
```
(프로젝트 A) - [BELONGS_TO] -> (Industry: 뷰티)
(프로젝트 A) - [BUILT_ON] -> (Platform: Cafe24)
(프로젝트 A) - [USES_TECH] -> (React, Node.js)
(프로젝트 A) - [COST] -> (3,500만원, 8주, 5명)
(프로젝트 A) - [WIN] -> (Win_Loss: 수주 성공, 사유: 경쟁사 대비 20% 저렴)
(프로젝트 A) - [HAS_DOCUMENT] -> (Document: 제안서)
(Document: 제안서) - [STORED_AS] -> (File: 제안서_v2.3.pdf)
```

### 3.2 RAG 시스템 (현실적 범위)

**✅ 1단계: 텍스트 검색 (필수)**
- 의미 기반 검색: "결제 취소 프로세스" → 관련 문서 섹션 정확 추출
- 임베딩: OpenAI Ada / Sentence Transformers

**✅ 2단계: 이미지 검색 (필수)**
- CLIP 모델로 디자인 시안 검색
- "다크모드 메인 비주얼" → 과거 제안서 이미지 반환

**⚠️ 3단계: 코드 검색 (선택)**
- 간단한 키워드 검색 수준
- "회원가입 이메일 인증" → 관련 JS 파일 검색

**❌ 제외: 동영상, 음성, 손글씨**
- 기술적 가능하지만 ROI 낮음
- 필요시 Phase 5 이후 검토

### 3.3 외부 데이터 통합 (최소화)

**✅ 트렌드 검색 (Web Search API)**
- 제안서 작성 시 "2025 웹 디자인 트렌드" 자동 검색
- 결과를 제안서 [최신 트렌드] 섹션에 참조

**❌ 경쟁사 자동 크롤링**
- 법적 이슈, 데이터 신뢰도 문제 → 제외
- 필요시 수동 입력

## 4. 주요 기능 (실행 가능한 것만)

### 4.1 영업팀: 수주 경쟁력 강화

#### **기능 1: 스마트 제안서 생성**

**입력:**
```
업종: 뷰티
타겟: 20대 여성
플랫폼: Shopify
예산: 3,000~3,500만원
```

**출력:**
```
[유사 프로젝트 레퍼런스]
1. B사 뷰티 브랜드 (2024.03, Shopify, 3,200만원)
   - 주요 기능: 라이브커머스, 회원등급제, AI 상품추천
   - 수주 성공, 사유: "AI 상품추천"이 차별점
   - 제안서 다운로드 링크

2. C사 화장품 몰 (2023.11, Shopify, 3,800만원)
   - 주요 기능: 정기구독, 포인트 적립, 리뷰 이벤트
   - 수주 실패, 사유: 견적 과다
   
[추천 기능]
- 라이브커머스 (3개 프로젝트에서 효과 검증)
- AI 피부 진단 (최신 트렌드, 2024 검색량 +340%)
- 플로팅 장바구니 (모바일 전환율 평균 +15%)

[예상 견적]
- 평균 견적: 3,400만원 (±12%)
- 공수: 8주
- 인력: PM 1명, 디자이너 1명, 개발자 2명, 퍼블리셔 1명
```

#### **기능 2: 자동 견적 산출**

**프로세스:**
1. 요구 기능 입력 (회원가입, 결제, 배송, 반응형)
2. 과거 유사 프로젝트 10건 분석
3. 평균 공수 계산
4. 견적 범위 제시 + 근거 데이터

**예시:**
```
입력: Shopify + 50개 상품 + 결제/배송 + 반응형
출력: 
- 견적: 3,500만원 (±15%)
- 근거: 과거 유사 프로젝트 8건 평균 3,420만원
- 공수: 8주 (범위: 7~9주)
- 리스크: Shopify API 연동 시 1주 버퍼 권장
```

#### **기능 3: Win/Loss 분석**

**저장 데이터:**
- 수주 성공/실패 여부
- 사유 (가격, 기술력, 일정, 기타)
- 경쟁사 정보 (선택)

**분석 결과 예시:**
```
[패턴 학습 결과]
- 공공 프로젝트: "보안 인증서" 포함 시 승률 +35%
- 스타트업: 유지보수 별도 제안 시 승률 -20%
- 뷰티 업종: "AI 기능" 강조 시 승률 +18%
```

**입력:**

3. 예약하기
4. 이벤트

[근거]
- 유사 F&B 프로젝트 5건 분석
- 평균 depth: 2.4

#### **기능 2: 기능정의서 자동 초안**

**프로세스:**
1. "회원가입 기능정의서 작성해줘" 요청
2. 과거 5개 프로젝트의 회원가입 정책 추출
3. 공통 항목 + 업종별 특이사항 통합
4. 초안 생성

**예시:**
```
[회원가입 기능정의서 초안]

1. 회원가입 방식
   - 이메일 가입 (필수)
   - 소셜 로그인 (카카오, 네이버, 구글)
   - 본인인증 (선택, 14세 미만 필수)

2. 필수 입력 정보
   - 이메일, 비밀번호, 이름, 휴대폰
   - 개인정보 수집 동의 (필수)
   - 마케팅 수신 동의 (선택)

3. 비밀번호 정책
   - 8자 이상, 영문+숫자+특수문자
   - 과거 3개 비밀번호 재사용 불가

[근거]
- 5개 프로젝트 공통 사항 통합
- 개인정보보호법 준수 (최신 기준)
```

#### **기능 3: Usecase 템플릿 제공**

**요청:** "결제 프로세스 Usecase"

**출력:**
```
[결제 프로세스 Usecase]

정상 시나리오:
1. 사용자가 상품 선택 → 장바구니 추가
2. 주문서 작성 (배송지, 요청사항)
3. 결제 수단 선택 (카드, 간편결제, 무통장)
4. 결제 승인
5. 주문 완료 → 이메일/SMS 발송

예외 시나리오:

- 결제 실패: 사유 표시 + 재시도 유도

[근거]
- 과거 커머스 프로젝트 8건 참조

### 4.5 PM: 프로젝트 관리

**입력:**
```

## 5. 구글과 차별화되는 추가 기능

### 5.1 AI 제안서 심사위원 (실행 가능)

**목적:** 제안서 작성 완료 후 AI가 품질 체크

**프로세스:**
1. 완성된 제안서 업로드
2. AI가 과거 성공 제안서 20건과 비교
3. 점수 + 개선 제안

**출력 예시:**
```
[제안서 품질 분석]

총점: 78/100

✅ 강점:
- 시각 자료 풍부 (이미지 15개, 평균 12개)
- 견적 근거 명확 (과거 프로젝트 3건 제시)

⚠️ 개선점:
- [차별화 전략] 섹션 짧음 (2페이지 vs 평균 5페이지)
- ROI 수치 부족 (성과 데이터 1개 vs 평균 3개)
- K사는 "보안" 키워드 선호 (현재 3회 vs 평균 8회)

📝 개선 제안:
1. [차별화 전략] 섹션에 "AI 기능" 추가 (최신 트렌드)
2. 과거 유사 프로젝트 성과 그래프 삽입
3. 보안 관련 내용 강화 (HTTPS, 개인정보 암호화)
```

**구현 방법:**
- LLM (GPT-4/Claude)로 제안서 텍스트 분석
- 과거 성공/실패 제안서 패턴 학습
- 키워드 빈도, 섹션 길이 등 정량 분석

```

```
[프로젝트 시뮬레이션 결과]

- 근거: 유사 프로젝트 6건 평균

⚠️ 리스크:
- 라이브커머스 기능: 테스트 기간 부족 (2주 추가 권장)
- AI 챗봇: 학습 데이터 준비 필요 (고객사 협조 필수)

**구현 방법:**

### 5.3 클라이언트 패턴 DB (실행 가능)

   - 일정 버퍼 20% 추가

**구현 방법:**
- 과거 이메일, 회의록 텍스트 분석
- 키워드 빈도 분석 (선호 표현 추출)
- 수작업 입력 + AI 보조

✅ 전자상거래법

- 사업자 정보 표시

- "의료광고법 준수: Before/After 이미지 미사용"

**구현 방법:**
